# 文献阅读 Agent — v1.0 项目汇报

**项目名称**：文献阅读 Agent（Literature Reading Agent）
**版本**：v1.0
**汇报日期**：2026年4月14日

---

## 一、项目概述

文献阅读 Agent 是一款面向学术研究场景的 AI 辅助工具，核心功能是将用户上传的学术论文 PDF 自动转化为演示文稿（PPT），并提供基于论文内容的智能问答服务。用户无需手动整理文献，即可快速获得可用于汇报的演示材料和深度问答支持。

**目标用户**：高校师生、科研人员、需要快速消化学术文献的从业者

---

## 二、核心功能

### 2.1 论文 → PPT 全自动生成
- 用户上传 PDF，选择 PPT 配置（模板风格、页数、语言、受众）
- 后端调用 Claude Sonnet 完整执行 paper-to-ppt 流程：论文深度解析 → 内容策划 → 逐页 SVG 生成 → 后处理 → 导出 PPTX
- 生成质量与人工使用 Claude Code CLI 手动操作一致
- **缓存机制**：相同 PDF + 相同配置直接复用已生成结果，无需重复调用 LLM

### 2.2 PPT 展示与演讲稿朗读
- 浏览器内展示生成的 SVG 幻灯片，支持键盘翻页
- 每页配套 AI 生成的演讲稿，支持一键 TTS 自动朗读并自动翻页
- 支持手动控制：暂停、停止、单页朗读

### 2.3 论文智能问答
- 基于 RAG（检索增强生成）技术，对论文内容进行段落级向量化索引
- 用户提问 → 检索相关段落 → LLM 生成回答，答案中自动标注页码引用 `(第N页)`
- 点击页码 badge 直接在页面内弹出 PDF 查看器，定位到对应页原文
- 支持语音输入提问（Chrome/Edge 浏览器）

### 2.4 PDF 原文查看器
- 内嵌 PDF.js 查看器，支持缩放、翻页、文字选择
- 从问答引用直接跳转到论文对应页，实现"答案 → 原文"的完整溯源链路

## 三、技术架构

```
┌─────────────────────────────────────────────────────┐
│                    前端 (React 18 + Vite)            │
│  UploadPage → ProgressPage → PptViewerPage + Chat   │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP / WebSocket
┌──────────────────────▼──────────────────────────────┐
│                  后端 (FastAPI + uvicorn)             │
│  /api/upload  /api/chat  /api/sessions  /ws/{id}    │
└──────┬──────────────────────────┬───────────────────┘
       │                          │
┌──────▼──────┐          ┌────────▼────────┐
│ PPT 生成    │          │  RAG 知识库      │
│ Claude CLI  │          │  BAAI/bge-m3    │
│ paper-to-ppt│          │  LlamaIndex     │
│ skill       │          │  GLM-4-Flash    │
└─────────────┘          └─────────────────┘
```

**技术栈**：
- 前端：React 18、TypeScript、Vite、Tailwind CSS v4、PDF.js
- 后端：Python 3.10、FastAPI、LlamaIndex、pypdf、python-pptx
- AI：Claude Sonnet（PPT生成）、GLM-4-Flash（问答）、BAAI/bge-m3（本地嵌入）
- 工具链：ppt-master skill（SVG生成）、paper-glance skill（论文解析）

## 四、开发过程

项目采用 OpenSpec 规范驱动开发，共完成 6 个迭代变更：

| 变更 | 内容 | 状态 |
|------|------|------|
| frontend-upload-ppt | 前端上传页面 + PPT 展示框架 | ✅ 完成 |
| presentation-chat | 左右分栏布局（PPT + 问答） | ✅ 完成 |
| oral-script-tts | 演讲稿生成 + TTS 朗读 + 自动翻页 | ✅ 完成 |
| voice-qa | 语音输入问答 + TTS 答案朗读 | ✅ 完成 |
| accurate-source-attribution | 问答引用精准溯源 + 行内页码 badge | ✅ 完成 |
| real-ppt-generation | 真实 PPT 生成流程 + 配置选择 + 缓存 | ✅ 完成 |

**关键技术决策**：
- PPT 生成从"API 模拟 Executor"改为"直接调用 Claude CLI"，质量提升显著
- RAG 从默认 token 切片改为段落级切片，检索准确率明显提升
- 问答来源从"展示 top-k 检索结果"改为"解析 LLM 实际引用页码"，消除噪声引用
- PDF 查看器从 iframe 原生 viewer 改为 PDF.js，解决页码定位不准问题

## 六、下一步计划

### 6.1 短期优化（v1.1）
- **PPT 生成进度可视化**：Claude CLI 运行时实时显示当前生成到第几页
- **错误恢复**：PPT 生成失败时保留已生成的部分，支持断点续生
- **移动端适配**：当前布局在小屏设备上显示不佳

### 6.2 中期功能（v1.2）
- **多论文联合问答**：上传多篇 PDF，跨文献检索与对比分析
- **PPT 在线编辑**：生成后支持修改文字、调整布局
- **导出增强**：支持导出 Word 版阅读笔记、思维导图

### 6.3 长期方向（v2.0）
- **知识库积累**：多次上传的论文自动建立个人知识库，支持跨论文检索
- **协作功能**：团队共享论文库，标注与评论同步
- **API 开放**：提供 REST API，支持第三方系统集成
