# 工作周报 · 文献阅读 Agent v1.0

**时间：** 2026 年 4 月第 3 周  
**项目：** 文献阅读 Agent — PDF 论文一键转 PPT + RAG 问答 Web 应用

---

## 一、项目概述

从零构建了一套面向学术场景的文献阅读辅助工具。用户上传一篇学术论文 PDF，系统自动完成两件事：

1. **生成 PPT 演示文稿** — 调用 Claude CLI 执行 `paper-to-ppt` skill，产出带演讲稿的学术汇报幻灯片
2. **构建 RAG 问答知识库** — 对 PDF 全文建立向量索引，支持用自然语言提问、引用溯源

整个过程在 Web 界面中实时可见，用户无需接触命令行。

---

## 二、技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React 18 + TypeScript + Vite + Tailwind CSS v4 |
| 后端 | Python FastAPI + uvicorn（conda 环境） |
| PPT 生成 | Claude CLI（`paper-to-ppt` skill）+ subprocess 异步调用 |
| RAG 检索 | LlamaIndex + BAAI/bge-m3 本地嵌入 |

| 语音 | 浏览器 Web Speech API（TTS 朗读 + 语音输入） |

## 三、本周完成工作

### 3.1 后端核心功能

**上传与会话管理**
- `POST /api/upload`：接收 PDF 文件 + PPT 配置参数（模板、页数、语言、风格、受众），创建 Session，异步启动任务
- 内存 Session Store：管理 `pending → processing → ready / error` 状态流转
- WebSocket `/ws/{session_id}`：实时向前端推送进度事件（`progress` / `done` / `error`）

**PPT 生成流水线（`ppt_generator.py` + `task_manager.py`）**
- 缓存机制：`cache_key = SHA256(PDF)[:16] + SHA256(config_json)[:8]`，同 PDF + 同配置直接命中缓存，秒级返回
- 调用 Claude CLI 以 `--print --dangerously-skip-permissions` 批处理模式运行 `paper-to-ppt` skill，40 分钟硬超时
- 实时日志：`subprocess.Popen` + 双线程分别流式读取 stdout / stderr，每行即时打印 `[CLAUDE-OUT]` / `[CLAUDE-ERR]`
- 结果移动：生成完成后将项目目录整体 `shutil.move` 到缓存目录，保证下次命中
- 预检：启动前验证 Claude CLI 可执行文件和 skill 文件是否存在，失败立即抛出明确错误

**RAG 索引（`rag_index.py`）**
- 自定义 PDF 解析：`pypdf` 逐页提取文字 → 检测并去除重复页眉 → 按段落分块（目标 1200 字符/块，最小 200 字符防碎片）
- 论文标题提取：优先读 PDF 元数据，回退到首页首行启发式识别
- 向量化：BAAI/bge-m3 本地模型，LlamaIndex `VectorStoreIndex`，持久化到磁盘
- 缓存：`SHA256(PDF)[:16]-v3` 作为目录名，已建索引直接复用

**RAG 问答（`chat.py`）**
- 检索 top-6 相关段落，拼接为带来源标注的上下文
- System prompt 强制要求每个论点附 `(第N页)` 页码标注
- 引用解析：从 LLM 回答中提取实际引用页码，过滤未被引用的检索片段，避免展示幻觉来源

**讲稿接口（`script.py`）**
- 优先从 PPT 项目 `notes/*.md` 文件加载演讲稿（skill 自动生成）
- `_clean_note()`：去除 `要点：` / `时长：` 元数据行，剥离 `[过渡]` 等方括号标注，仅保留正文
- 回退：若 notes 不存在，调用 LLM 根据 PPTX 内容自动生成讲稿

**幻灯片展示接口（`sessions.py`）**
- `GET /slides` — 列出 `svg_final/*.svg` 文件列表
- `GET /slides/{filename}` — 逐张返回 SVG 图片（防路径穿越处理）
- `GET /ppt` — 下载 PPTX 文件
- `GET /pdf` — 内联返回原始 PDF

### 3.2 前端核心功能

**上传页（`UploadPage.tsx`）**
- 拖拽 / 点击上传 PDF，上传前预检格式
- PPT 配置面板：7 种模板风格（学术答辩 / Anthropic / Google / 麦肯锡 / Exhibit / 重庆大学 / 自由设计）、页数、语言、风格、受众
- 配置作为最终确认直接进入生成，不再二次交互询问

**进度页（`ProgressPage.tsx`）**
- 双进度条实时展示 PPT 生成和知识库构建两个并行任务
- WebSocket 连接断开自动重连，`done` 事件到达后 1.2 秒自动跳转
- **HTTP 轮询兜底**：每 8 秒主动查询 Session 状态，防止 WebSocket 事件丢失导致页面永久卡住

**PPT 展示页（`PptViewerPage.tsx`）**
- SVG 幻灯片逐张渲染，左右键翻页
- 讲稿面板：可展开 / 折叠，显示当前页演讲稿
- **连续播报**：点击"▶ 连续播报"按钮，TTS 自动朗读当前页讲稿，读完后翻页继续播报
- PPTX 一键下载

**问答面板（`ChatPanel.tsx`）**
- 对话式问答界面，Shift+Enter 换行，Enter 发送
- 内联引用：回答中的 `(第N页)` 渲染为可点击的跳转按钮，点击打开原文 PDF 对应页
- **TTS 自动朗读**：开启后每次回答自动朗读，支持随时停止
- **语音输入**：Chrome / Edge 下支持点击麦克风图标说话提问

### 3.3 TTS 文本预处理（`utils/tts.ts`）

为解决演讲稿 TTS 朗读质量问题，实现了统一的预处理函数 `preprocessForTts()`，在所有 `speechSynthesis.speak()` 调用前执行：

1. **去除舞台指令**：正则删除所有 `[停顿]` `[过渡]` `[强调]` 等方括号标注，这些是写给演讲者看的提示，不应被朗读
2. **数学符号口语化**：
   - `M_{t+1}` → "M t加1"（花括号下标转自然语言）
   - `m_i` → "m i"（下划线下标拆分，不再读"m下划线i"）
   - `g_write` → "g write"（变量名下划线拆分）
   - `x^2` → "x平方"，`x^3` → "x立方"
   - `x^{n+1}` → "x的n加1次方"

## 四、关键 Bug 修复

| # | 问题 | 根因 | 修复方式 |
|---|------|------|----------|
| 1 | Claude CLI 找不到 skill，PPT 生成失败 | `REPO_ROOT` 计算错误，`.parent.parent` 指向 `.claude/` 而非项目根目录 | 改为 `.parent.parent.parent`，并加预检日志 |
| 2 | 同 PDF + 同配置不命中缓存，每次重新生成 | Claude 将项目写到 `projects/{name}/` 但缓存检查路径不同；缓存目录和输出目录不统一 | 生成后将项目整体 `shutil.move` 到缓存目录；统一 `PPT_CACHE_DIR` 为 `E:\agent\reading agent\projects` |
| 3 | 前端进度页卡在 85%，不跳转 | WebSocket `done` 事件在 0 个客户端时发出，前端未收到 | 增加 HTTP 轮询兜底（每 8 秒），任意一路触发即可跳转 |
| 4 | Claude CLI 日志不可见，无法定位问题 | 使用 `subprocess.run` 等待结束才输出 | 改为 `subprocess.Popen` + 两个读取线程，实时流式打印每一行输出 |
| 5 | 演讲稿 TTS 读出 `[停顿]`、`m下划线i` | 原始文本未处理直接传给 `speechSynthesis` | 新增 `preprocessForTts()` 预处理 |
